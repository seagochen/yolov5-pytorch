## 后端架构

### 分层设计

```
backend/
├── main.py              # 应用入口，CORS 配置
├── api/                 # API 层
│   ├── router.py        # 路由聚合
│   └── endpoints/       # 具体端点
│       ├── divination.py    # 占卜相关 API
│       └── interpretation.py # AI 解卦 API
├── services/            # 业务逻辑层
│   ├── gua_service.py       # 卦象计算服务
│   └── gemini_service.py    # AI 服务
├── core/                # 核心模块
│   └── models.py            # 数据模型
├── config/              # 配置模块
│   ├── settings.py          # 配置管理
│   └── gemini_client.py     # Gemini 客户端
└── prompts/             # AI 提示词
    └── interpretation_prompts.py
```

### 数据流

1. **占卜流程**
   ```
   用户请求 → API 端点 → GuaService 计算卦象 → 加载 JSON 数据 → 返回结果
   ```

2. **AI 解卦流程**
   ```
   占卜结果 + 问题 → API 端点 → GeminiService → 构建 Prompt → 调用 AI → 返回解读
   ```

### 状态管理

使用 React useState 管理本地状态：

- `question`: 用户输入的问题

- `loading` / `interpreting`: 加载状态
- `error`: 错误信息

## 卦象计算逻辑

### 数字转卦象

1. 六个数字（6-9）表示六爻
2. 每个数字含义：
   - 6: 老阴（变爻）
   - 7: 少阳
   - 8: 少阴
   - 9: 老阳（变爻）

3. 转换为二进制：
   - 阳爻 (7, 9) → 1
   - 阴爻 (6, 8) → 0

4. 计算本卦和变卦的卦序号

### 随机占卜

模拟三枚硬币投掷：
- 随机生成 6 个数字（6-9）
- 根据数字计算卦象

## API 设计

### RESTful 规范

- `POST /api/divination/calculate`: 根据数字计算卦象
- `POST /api/divination/random`: 随机占卜
- `GET /api/divination/gua/{index}`: 获取卦象信息
- `POST /api/interpretation/analyze`: AI 解卦

详见 [API 参考](03-API-REFERENCE.md)。

## 前端组件架构

### Tab 系统设计

系统采用三个独立的 Tab 组件，提供完整的易学功能：

```
App.tsx (主应用)
├── 导航栏 (固定顶部)
├── DivinationTab (周易占卜)
├── BaziTab (八字排盘)
└── QimenTab (奇门遁甲)
```

### 1. 周易占卜 (DivinationTab)

**功能**：
- 随机占卜或手动输入数字
- 显示本卦和变卦
- AI 智能解卦

**状态管理**：
- `question`: 用户问题
- `divinationResult`: 占卜结果
- `interpretation`: AI 解读
- `loading` / `interpreting`: 加载状态

### 2. 八字排盘 (BaziTab)

**功能**：
- 输入姓名、出生年月日时分
- 支持公历/农历选择（目前仅支持公历）
- 显示四柱（年月日时）
- 五行分析（含统计图）
- 神煞分析（吉神/凶神）

**核心特性**：
- 公历转农历功能
- 四柱计算（天干地支）
- 五行统计和可视化
- 神煞自动判定

### 3. 奇门遁甲 (QimenTab)

**功能**：
- 输入公历年月日时分
- 九宫格排盘布局
- 每宫显示：八卦、地盘、天盘、八门、九星、八神
- 吉凶判断和值符标识
- 用神含义显示
- 整体分析（最佳/不利方位、建议）
- 使用指南弹窗

**用神对应关系**：
- 年干（坎宫1）：求测者本人、自己的想法
- 月干（震宫3）：兄弟姐妹、同事、朋友
- 日干（乾宫6）：父母、领导、上级、贵人
- 时干（兑宫7）：子女、下属、晚辈

**视觉设计**：
- 宫位边框颜色对应吉凶
- 值符宫位特殊标识
- 用神说明区块独立显示

## 数据架构设计

### 设计理念

为了提高系统的可维护性和扩展性，采用**数据与逻辑分离**的架构：

- 所有说明性文字和定义数据存储在 JSON 文件中
- 代码只负责展示逻辑
- 前端通过 API 动态加载数据

### 数据存储结构

```
backend/data/
├── gua/                    # 64卦象数据
│   ├── 000000_001001.json
│   └── ...
├── qimen_guide.json        # 奇门遁甲指南
├── bazi_guide.json         # 八字排盘指南
└── README.md               # 数据说明文档
```

### 数据文件说明

#### 1. qimen_guide.json

包含奇门遁甲的完整指南数据：

- **基本介绍**：概念和背景
- **九宫格元素**：地盘、天盘、八门、九星、八神的详细说明
- **四柱用神**：
  - 年干、月干、日干、时干的定义
  - 对应宫位映射（坎1、震3、乾6、兑7）
  - 实际应用举例（求职、求财、问子女、问自己）
- **八卦宫位**：九宫的方位、五行、象征意义
- **用神宫位映射**：每个宫位的用神含义详解
- **吉凶判断标准**：吉门/凶门、吉星/凶星、吉神/凶神、三奇
- **解读步骤**：五步解盘法

#### 2. bazi_guide.json

包含八字排盘的完整指南数据：

- **基本介绍**：八字的概念和作用
- **四柱说明**：年柱、月柱、日柱、时柱的含义和时间段
- **十天干**：甲乙丙丁戊己庚辛壬癸的五行属性和特征
- **十二地支**：子丑寅卯等的生肖、时辰、季节对应
- **五行分析**：
  - 木火土金水的属性、特征、对应关系
  - 相生相克规律
  - 五行平衡说明
- **神煞**：吉神和凶神的列表和含义
- **历法系统**：公历、农历、节气的区别和应用
- **使用建议**：输入要点和注意事项

### 数据访问层

#### API 端点

```
GET /api/guide/qimen   # 获取奇门遁甲指南
GET /api/guide/bazi    # 获取八字排盘指南
GET /api/guide/        # 获取所有指南
```

#### 实现位置

`backend/api/endpoints/guide.py` - 指南数据 API 端点

#### 前端数据加载

```typescript
// 动态加载指南数据
import { getQimenGuide, getBaziGuide } from '../api/client';

const guide = await getQimenGuide();
// 使用 guide 数据渲染组件
```

### 架构优势

#### 1. 数据与逻辑分离
- 说明文字集中管理，便于统一修改
- 代码更简洁，只负责展示逻辑
- 降低维护成本

#### 2. 易于扩展
- 添加新内容只需修改 JSON 文件
- 无需修改代码即可更新说明
- 支持版本控制和协作

#### 3. 国际化友好
- 数据结构化便于翻译
- 可轻松添加多语言支持
- 示例：`qimen_guide.zh-CN.json`、`qimen_guide.en.json`

#### 4. 性能优化
- 前端可缓存 API 数据
- 减少代码体积
- 按需加载

#### 5. 可测试性
- 数据可独立验证
- 便于单元测试和 E2E 测试

### 数据维护流程

#### 后端开发者
1. 修改数据文件：编辑 `backend/data/*.json`
2. 验证格式：`python -m json.tool qimen_guide.json`
3. 重启服务：数据立即生效

#### 前端开发者
1. 使用 API 客户端获取数据
2. 根据返回的 JSON 结构渲染界面
3. 降级处理：API 失败时使用本地默认值

#### 内容维护者
1. 直接编辑 JSON 文件
2. 无需了解代码
3. 修改后通知开发者重启服务
